# Cookbook

Practical recipes and patterns for common streamz use cases.

## Recipes

Recipes are organised by domain:

### Data Processing
- Batch processing with error recovery
- Parallel enrichment pipelines
- Rate-limited API calls

### Event Handling
- Real-time event aggregation
- Session windowing for user activity
- Deduplication strategies

### Integration Patterns
- Database batch inserts
- Message queue consumption
- HTTP request fan-out

## Contributing Recipes

When adding a recipe:

1. Focus on a single, concrete problem
2. Provide complete, runnable code
3. Explain the "why" behind design choices
4. Include error handling
5. Add performance considerations where relevant
